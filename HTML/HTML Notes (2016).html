<head> elements:

  • Container for metadata (data about data) and is placed between the <html> tag and the <body> tag
  • Metadata typically define the document title, character set, styles, links, scripts, and other meta information

<title> element:

  • Defines the title of the HTML document
  • Provides a title for the page when it is added to favorites
  • Displays a title for the page in search engine results

<style> element:

Defines style information for a single HTML page



<link> element:

Used to link to external style sheets



<meta> element:

  • Used to specify which character set is used, page description, keywords, author, and other metadata
  • Used by browsers (how to display content), by search engines (keywords), and other web services
Description of attribute Example
Define the character set used <meta charset="UTF-8">
Define a description of your web page <meta name="description" content="Free Web tutorials">
Define keywords for search engines <meta name="keywords" content="HTML, CSS, XML, JavaScript">
Define the author of a page <meta name="author" content="Kieron Lee">
Refresh document at a certain time interval <meta http-equiv="refresh" content="30">

<base> element:

Specifies the base URL and base target for all relative URLs in a page



Example:

 <base href="http://www.w3schools.com/images/" target="_blank">

Images:

Syntax:

 <img src="(Image URL)" alt="(Alternate text)" height="(Height of image in pixels/percent)" width="(Width of image in pixels/percent)">

Example:

<img src="facebook.com" alt="Image of FB" style="width:300px;height:230px;">
OR <img src="html5.gif" alt="HTML5 Icon" width="128" height="128"> (Avoid using this)

Result:




Attributes:

  • src attribute specifies the URL (web address) of the image
  • alt attribute specifies an alternate text for an image if the image cannot be displayed or viewed and can be read by screen readers

Images stored in other locations:

Example of images stored in another folder:

 <img src="/images/html5.gif" alt="HTML5 Icon" style="width:128px;height:128px;">
  • Browser expects to find image in same folder as webpage
  • Include the folder name in the src attribute

Example of images stored on another server:

 <img src="http://www.w3schools.com/images/green.jpg" alt="W3Schools">

Animated images/GIFs:

Example of GIF Image:

 <img src="programming.gif" alt="Computer Man" style="width:48px;height:48px;">

Result:




Image floating:

Use CSS to float an image to the left or right of text



Syntax:

 <... style="float:(left/right);...">

Example of floated images:

 <p><img src="smiley.gif" alt="Smiley face" style="float:right;width:42px;height:42px;">The image will float to the right of the text.</p>
OR                                          <p><img src="smiley.gif" alt="Smiley face" style="float:left;width:42px;height:42px;">The image will float to the left of the text.</p>

Result:


The image will float to the right of the text.

The image will float to the left of the text.


Image maps:

  • Images with clickable areas
  • Use the <map> tag to define an image map
  • The <map> tag is associated with the <img>'s usemap attribute and creates a relationship between the image and the map
  • The <map> tag contains a number of <area> tags that defines the clickable areas in the image-map
  • When clicked, the images redirects the user to another URL or image depending on the href value

Example of image map:

		<img src="planets.gif" alt="Planets" usemap="#planetmap" style="width:145px;height:126px;">
		<map name="planetmap">
			<area shape="rect" coords="0,0,82,126" alt="Sun" href="sun.htm">
			<area shape="circle" coords="90,58,3" alt="Mercury" href="mercur.htm">
			<area shape="circle" coords="124,58,8" alt="Venus" href="venus.htm">
		</map>

Coordinate values:

Value Description
x1, y1, x2, y2 Coordinates of the left, top, right, bottom corner of a rectangle (for shape="rect")
x, y, radius Coordinates of the center of a circle and the radius (for shape="circle")
x1, y1, x2, y2, xn, yn Coordinates of the edges of a polygon. First and last coordinate pair should be the same. (for shape="poly")

Links:

Syntax:

 <a href="(Link)">(Link Description)</a>

Example:

 <a href="http://www.w3schools.com/html/">Link to Facebook</a>

Link to MouseHunt
  • href attribute specifies the destination address
  • Link text can be an HTML image or any other HTML element
  • Use trailing slash (/) on subfolder addresses

Local links:

  • Links to the same web site
  • Specified with a relative URL (without http://www....)

Example:

 <a href="html_images.asp">HTML Images</a>

Links (Colour):

Property of links:

 When link is moused over, cursor turns into hand and colour of link element changes

By default:

  • Unvisited links are underlined and blue
  • Visited links are underlined and purple
  • Active links (Cursor being held down over link) are underlined and red

Change the default colours by using CSS styles


Example:

		<style>
		a:link {
		color:green;
		background-color:transparent; 
		text-decoration:none;
		}
		a:visited {
		color:pink;
		background-color:transparent;
		text-decoration:none
		}
		a:hover {
		color:red;
		background-color:transparent;
		text-decoration:underline;
		}
		a:active {
		color:yellow;
		background-color:transparent;
		text-decoration:underline;
		}
		</style>
Text decoration values

Target attribute:

Specify where linked document will be opened


Example:

<a href="http://www.w3schools.com/" target="_blank">Visit W3Schools!</a>

Target Value: Description
_blank Opens linked document in new window or tab
_self (Default) Opens linked document in the same page
_parent Opens linked document in the parent frame
_top Opens linked document in the full body of the window
framename Opens linked document in a named frame

Use target="_top" to break out of the frame if page is locked in a frame

Bookmarks:

  • Allows reader to jump to specific part of webpage
  • Practical for long webpages

Steps to create bookmark:

  1. Create a bookmark with the id Attribute
    • <h2 id="tips">Useful Tips Section</h2>
  2. Add a link to the bookmark from the same page:
    • <a href="#tips">Visit the Useful Tips Section</a>
  3. Add a link to the bookmark from another page:
    • <a href="html_tips.htm#tips">Visit the Useful Tips Section</a>

  • rowspan="(Number of rows spanned)" for spanning rows

Blocks:

Block-level elements:

Block-level elements always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can)



Examples of block-level elements:
  • <div>
  • <h1> - <h6>
  • <p>
  • <form>

Inline elements:

Inline elements does not start on a new line and only takes up as much width as necessary



Examples of inline elements:
  • <span>
  • <a>
  • <img>

<div>:

  • Defines a section in a document (block-level)
  • Used as a container for other HTML elements
  • Has no required attributes, but both style and class are commonly used
  • Can be used together with CSS to style blocks of content

Example:

		<div style="background-color:black;color:white;padding:20px;">
		<h2>London</h2>
		<p>London is the capital city of England. It is the most populous city in the United Kingdom, with a metropolitan area of over 13 million inhabitants.</p>
		</div>

Result:

London

London is the capital city of England. It is the most populous city in the United Kingdom, with a metropolitan area of over 13 million inhabitants.



<span>:

  • Defines a section in a document (inline)
  • Often used as a container for some text
  • Has no required attributes, but both style and class are commonly used
  • Can be used together with CSS to style parts of text

Example:

		<h1>My <span style="color:red">Important</span> Heading</h1>

Result:

My Important Heading

Layouts:

Layout elements:

  • Websites often display content in multiple columns (like a magazine or newspaper)
  • HTML5 offers new semantic elements that define the different parts of a web page
  • <header> - Defines a header for a document or a section
  • <nav> - Defines a container for navigation links
  • <section> - Defines a section in a document
  • <article> - Defines an independent self-contained article
  • <aside> - Defines content aside from the content (like a sidebar)
  • <footer> - Defines a footer for a document or a section
  • <details> - Defines additional details
  • <summary> - Defines a heading for the <details> element

Layout techniques:

CSS frameworks:

Create layouts quickly by using frameworks such as W3.CSS or Bootstrap



CSS floats:

Advantages:

  • Easy to learn
  • Bugs are well aware and fixes are well documented

Disadvantages:

  • Floating elements are tied to the document flow, which may harm the flexibility
  • No vertical centering
  • No equal heights

Use for: Large layout blocks that don't need equal heights and vertical centering



Example:

		<html>
		<head>
		<style>

		width: 100%;

		background-color: black;
		clear: left;
		text-align: center;
		}

CSS flexbox:

Advantages:

  • Ensures that elements behave predictably when the page layout must accommodate different screen sizes and different display devices
  • Vertical centering
  • Equal heights

Disadvantages:

  • Does not work in IE10 and earlier
  • Complex to understand and syntax is unintuitive

Use for: App layouts where things need to stretch and squish

		padding: 15px;
		-webkit-flex: 1 100%;
		flex: 1 100%;
		}

		.article {
		text-align: left;
		}

		header {background: black;color:white;}
		footer {background: #aaa;color:white;}
		.nav {background:#eee;}

		.nav ul {
		list-style-type: none;
		padding: 0;
		}
		.nav ul a {
		text-decoration: none;
		}

		@media all and (min-width: 768px) {
		.nav {text-align:left;-webkit-flex: 1 auto;flex:1 auto;-webkit-order:1;order:1;}

Used to define a style for a class of elements


Syntax:

		(Unique tag).(Unique class) {

Notes:

  • Use lowercase for elements and attributes
  • No spacing between tags and displayed text
  • Use HTML headings for headings only, not to bold or enlarge text
  • Use American spelling
  • Use single/double quotation marks depending on situation
    <p title='John "ShotGun" Nelson'> OR <p title="John 'ShotGun' Nelson">
  • HTML head contains meta data which is not displayed
  • Formatting (TL;DR - Try to use <em> and <strong> as they are semantic. Text to speech and mobile browsers can’t detect <b> and <i>)
  • Use blockquotes for long quotations (Paragraphs and sections). Blockquote elements are usually indented
  • <address> elements are italicised. Most browsers will add a line break before and after the element.
  • Bi-Directional Override is to override the current text direction. Use attribute value of either ‘ltr’ or ‘rtl’.
